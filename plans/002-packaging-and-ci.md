## Why this matters

`typescript` is declared as a **peerDependency** of a published CLI. npm v7+ auto-installs peer dependencies, so every `npm install -g turso-introspect` drags the entire TypeScript compiler onto end users' machines for no reason — TypeScript is purely a build/typecheck-time tool here. It belongs in `devDependencies`. Separately, the repo has no CI at all: biome, typecheck, and the (new, from plan 001) test suite only run when someone remembers to run them. A small GitHub Actions workflow makes every push/PR self-verifying, which the other plans in this series rely on.

## Steps

### Step 0: Restore the `export` on `ForeignKey` in `src/lib/schema.ts`

Commit `1bbd6e0` removed `export` from `interface ForeignKey` (`src/lib/schema.ts:13`), but the exported `Table` interface embeds `foreignKeys: ForeignKey[]` and `src/lib/formatter.test.ts:3` imports the type. This breaks `tsc --noEmit` on main. Change line 13 from `interface ForeignKey {` to `export interface ForeignKey {`. Touch nothing else in the file.

Commit separately: `fix: re-export ForeignKey type referenced by exported Table interface`.

**Verify**: `git diff src/lib/schema.ts` shows exactly one changed line (the `export` keyword added).

### Step 1: Move typescript from peerDependencies to devDependencies

In `package.json`: delete the entire `peerDependencies` block; add `"typescript": "^6.0.2"` to `devDependencies` (keep alphabetical order). Run `bun install`.

**Verify**: `bun install` → exit 0; `node -e "const p=require('./package.json'); if (p.peerDependencies) process.exit(1); if (!p.devDependencies.typescript) process.exit(1)"` → exit 0 (or check by reading the file).

### Step 2: Confirm the typecheck script now works

**Verify**: `bun run typecheck` → exit 0, no output. If it reports type errors in `src/`, STOP (see STOP conditions).

### Step 3: Add the CI workflow

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  ci:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest
      - run: bun install --frozen-lockfile
      - run: bunx biome check src
      - run: bun run typecheck
      - run: bun test
      - run: bun run build
```

Note the lint step is `bunx biome check src` (scoped to `src`), not `bun run check` — the repo's `check` script lints `.` which can trip over local-only files.

**Verify**: `bun install --frozen-lockfile && bunx biome check src && bun run typecheck && bun test && bun run build` run locally in that order → all exit 0 (this mirrors exactly what CI will do).
