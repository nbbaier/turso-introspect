## Why this matters

The `diff` subcommand compares the **full** formatted SQL of two sources — including the generated header, which contains `-- Database: <name>` and `-- Timestamp: <ISO>` lines. Two different databases always have different `Database:` lines, and two introspections always have different timestamps. Consequence: `diff` can essentially **never** print "Schemas are identical", and every diff output is polluted with header noise. The core promise of the subcommand (tell me whether two schemas match) is broken. The fix is to strip the generated header from both inputs before comparing.

## Steps

### Step 1: Add a header-stripping helper to `diff.ts`

Add (and `export` it, so tests can target it directly):

```ts
export function stripGeneratedHeader(sql: string): string {
	const lines = sql.split("\n");
	let start = 0;
	while (start < lines.length) {
		const line = (lines[start] ?? "").trim();
		const isHeaderLine =
			line === "" ||
			line.startsWith("-- Schema generated by turso-introspect") ||
			line.startsWith("-- Database:") ||
			line.startsWith("-- Timestamp:");
		if (!isHeaderLine) break;
		start++;
	}
	return lines.slice(start).join("\n");
}
```

This removes only the *leading* block of blank/header lines; a `-- Database:` string later in the body (e.g. inside a trigger) is untouched.

**Verify**: `bun run typecheck` (if available) or `bun test` → no new failures.

### Step 3: Tests

Create `src/commands/diff.test.ts` (unit-test `stripGeneratedHeader`; testing the full `diff()` function requires live sources, so target the helper plus one composed case):

- Header + body → header removed, body intact (first line of result is the first non-header line).
- Input with no header (plain SQL) → returned unchanged.
- Input that is *only* a header → empty string.
- `-- Database: x` appearing mid-body (after real SQL) → preserved.
- Composed equality case: build two strings that differ only in `-- Database:`/`-- Timestamp:` header lines; assert `stripGeneratedHeader(a) === stripGeneratedHeader(b)`.
