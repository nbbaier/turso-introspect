## Why this matters

Turso's audience is overwhelmingly TypeScript developers, and the schema data the tool already extracts (`Column.type`, `notnull`, `pk`) is exactly what's needed to emit row-type interfaces ŌĆö no new introspection required, purely a presentation layer. This gives users compile-time-checked queries with `@libsql/client` without adopting an ORM. Scoped as an MVP: plain interfaces plus a `Tables` index type. Deliberately NOT a Drizzle/Kysely competitor ŌĆö no relations, no branded types, no runtime validators.

## Type-mapping spec (the contract ŌĆö implement exactly)

SQLite type affinity rules (SQLite docs ┬¦3.1), applied to the **uppercased** declared type:

1. contains `INT` ŌåÆ `number`
2. else contains `CHAR`, `CLOB`, or `TEXT` ŌåÆ `string`
3. else contains `BLOB` or is empty string ŌåÆ `Uint8Array`
4. else contains `REAL`, `FLOA`, or `DOUB` ŌåÆ `number`
5. else (NUMERIC affinity: `DECIMAL`, `BOOLEAN`, `DATE`, etc.) ŌåÆ `number | string`

Nullability: append `| null` when `notnull === 0 && pk === 0`. (`INTEGER PRIMARY KEY` is the rowid and never null; non-INTEGER pk columns in SQLite can technically hold NULL, but treating declared PKs as non-null is the pragmatic contract ŌĆö document it in the generated header comment.)

Naming: interface name = table name ŌåÆ split on non-alphanumerics ŌåÆ PascalCase each part ŌåÆ join (`user_posts` ŌåÆ `UserPosts`; leading digit ŌåÆ prefix `T`, e.g. `2fa_codes` ŌåÆ `T2faCodes`). Property names: emit as-is when they match `/^[A-Za-z_$][A-Za-z0-9_$]*$/`, otherwise quote (`"created at": string`).

Output shape:

```ts
// Generated by turso-introspect
// Database: <name>
// Timestamp: <ISO>
// Note: columns declared PRIMARY KEY are emitted as non-nullable.

export interface User {
	id: number;
	email: string;
	created_at: string | null;
}

export interface Tables {
	users: User;
}
```

`Tables` maps original table names (quoted if non-identifier) to interfaces, in the same table order as the schema. Views are omitted in the MVP (no column data is introspected for views ŌĆö `Schema.views` only carries `sql`).
